What this bot does

The short version, no tech speak

⏰

9:55 AM Mon–Thu: Your Mac automatically wakes up — as long as it's plugged into power. macOS ignores scheduled wake events on battery, regardless of whether the lid is open or closed

🤖

10:00 AM: The bot runs silently in the background — no popup, no browser window

🔑

Login: The bot signs in with your UC Davis credentials automatically. Your password is never in any file — only in your Mac's secure Keychain

📱

Duo Push (first time only): The very first run asks for a Duo Push on your phone. After you approve it once, the session is saved and you'll never need to do it again

✅

Booking: The bot tries Table 7 first. If it's fully booked, it automatically tries Table 6, then 8, then 5, then 9 — spiraling outward until it finds something available. If the first attempt fails, it retries once before giving up.

📅

Calendar invite: On success, a Google Calendar invite is automatically sent to you and your org's shared inbox (set as INVITE_TO in index.js) with the booked table and time slot

📋

Log + alert: Result is saved to booking.log — and if it failed, you'll get an email automatically

📋

Before you start

Everything you need ready

  • A Mac that you can leave plugged in overnight
  • Your UC Davis username and password (same ones for MyUCDavis)
  • Your phone nearby for a one-time Duo Push
  • A Gmail address optional — only needed if you want email alerts when booking fails

Setup

Do this once — takes about 5–10 minutes

1

Download the bot folder from Google Drive

Find the shared Google Drive folder and download it. When it downloads, you'll get a .zip file — double-click it to unzip. Then move the mu-booking-bot folder to your home folder (the one with your name in Finder's sidebar).

💡

The easiest way to move it: open a new Finder window, press ⌘ + Shift + H to go to your home folder, then drag the mu-booking-bot folder in there.

2

Open Terminal

Press ⌘ + Space, type Terminal, and press Enter. A window with a blinking cursor will open — that's it.

3

Install Homebrew

Homebrew is a free tool that makes installing software on a Mac simple. Paste this entire line into Terminal and press Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your Mac password (won't show as you type — that's normal). The install takes a couple of minutes. When it finishes you'll see a prompt again.

⚠️

Already have Homebrew? If you've used it before, just skip this step. If you're not sure, run brew --version — if it prints a version number, you're good.

4

Install Node.js

Node.js is what runs the bot. Once Homebrew is installed, type this and press Enter:

brew install node

Takes about a minute. When it finishes, confirm it worked by running:

node --version

You should see something like v22.0.0 — any version number means you're good.

5

Go to the bot folder

Type this and press Enter:

cd ~/mu-booking-bot

Nothing visible happens — that's normal. You're now inside the bot folder.

6

Run the setup script

Type this and press Enter:

bash setup.sh

It will walk you through everything step by step. Here's what it will ask:

  • Your UC Davis username (e.g. jsmith)
  • Your UC Davis password — won't show as you type, that's normal
  • An email address to send failure alerts to (optional — press Enter to skip)
  • If you want email alerts: a Gmail address to send from and a Gmail App Password (see the box below)
📧

How to get a Gmail App Password (only needed if you want email alerts):

  • Go to myaccount.google.com
  • Click Security → 2-Step Verification (must be enabled first)
  • Scroll down and click App passwords
  • Create a new one, name it mu-booking-bot
  • It gives you a 16-character password — copy it and paste it when setup.sh asks
7

Set your Mac to wake at 9:55 AM

The bot can't wake your Mac on its own — macOS has to do it. Open Terminal and run this command:

sudo pmset repeat wake MTWR 09:55:00

It will ask for your Mac password (won't show as you type — that's normal). Once done, your Mac will automatically wake at 9:55 AM every Monday through Thursday.

💡

Lid open or closed — doesn't matter. What matters is that your Mac is plugged into power. macOS ignores scheduled wakes on battery regardless of lid state. As long as it's plugged in, it wakes silently, books the table, then goes back to sleep.

8

Fill in your blackout dates

Open the file exceptions.json in the bot folder. You need to decide which dates you personally don't want the bot running.

Things to consider adding:

  • The 4th to last week of the quarter until the new quarter date drop
  • Any additional breaks
  • The quarter registration opening day — see the important note below
📅

Important: The start of each quarter (manual booking required)

At the start of each quarter, the MU releases the first 4 weeks of table availability all at once on a single registration day. These drop dates are listed at:

→ memorialunion.ucdavis.edu/reservations

How it works — Spring 2026 as an example:

  • Registration opens: Monday, March 23 at 10 AM
  • On that day, all Mon–Fri tables from March 30 through April 20 become available at once — that's the entire first 4 weeks
  • You must book these manually and quickly — it's competitive, and the bot only books one date at a time so it won't handle this
  • Starting Tuesday March 24, the bot takes over automatically: it books April 21. Wednesday March 25 → books April 22. Thursday March 26 → books April 23. One day at a time, 4 weeks out, from that point forward

What to add as an exception:

Add only the registration opening day as a blackout date (e.g. start: 2026-03-23 end: 2026-03-23). The bot doesn't need to run that day — you're handling it manually. The very next morning (Tuesday) the bot resumes and picks up exactly where you left off.

⚠️ Do not block Tue–Thu of that opening week as exceptions — the bot needs to run those days to book their 4-weeks-out dates.

See the Blackout Dates section below for step-by-step instructions on how to edit this file.

9

Do a test run — and approve Duo once

Run the bot right now to confirm everything works:

node index.js

Watch the Terminal output as it runs. When a Duo Push appears on your phone, approve it within 60 seconds. This is the only time you'll ever need to do this — the session is saved automatically after this run.

🎉

Setup complete! The bot will run every weekday at 10 AM from here on. Nothing else to do.

What happens every day at 10 AM

What to expect once it's running

The bot runs completely silently — no popup, no browser window. Check booking.log afterward to confirm it ran and what it booked.

➕

Adding blackout dates the easy way

No file editing required

Instead of editing exceptions.json by hand, you can run this command and just answer the prompts:

npm run add-exception

It will ask you for a label, a start date, and an end date — then add it automatically and sort the file. You can still edit the file manually if you prefer.

🗓

Blackout dates

Telling the bot when NOT to run

Open the file exceptions.json in any text editor (TextEdit, Notes, VS Code — whatever you have). Add any dates where you don't want or need a table.

Each entry looks like this:

// Copy this block to add a new date range: { "label": "First Two Weeks – Fall 2026", "start": "2026-09-24", ← year-month-day "end": "2026-10-07" ← both days included }
📌

Date format is strict: always YYYY-MM-DD

  • ✅ Correct: 2026-09-24
  • ❌ Wrong: Sep 24, 2026 or 9/24/26

To remove a date range: delete the entire { ... } block for that entry.

⚠️

After deleting an entry, make sure the last remaining entry in the list does not have a comma at the end of it. A leftover comma will cause the bot to fail to read the file.

Save the file when done. No restart needed — the bot re-reads it every time it runs.

🔴

Remember to add the quarter registration opening day as an exception each quarter. On that day, the first 4 weeks of tables are released all at once at memorialunion.ucdavis.edu/reservations — book those manually. Only block that one day; the bot picks up the next morning and runs normally from there.

Duo Push — only needed once

After the first run, it's fully automatic

1️⃣

First run only: A Duo Push appears on your phone. Approve it within 60 seconds. The bot waits for you.

✅

Running it manually

For testing or booking outside the schedule

Two ways to run manually — pick whichever you prefer. Both do the same thing.

Option A — Terminal

Type a command. More control over which mode to run.

Option B — One-click launcher

Set up once, then just double-click an icon — no Terminal ever again.

Option A — Terminal

1

Open Terminal

Press ⌘ + Space on your keyboard, type Terminal, then press Enter. A window with a blinking cursor will open.

2

Go to the bot folder

Click inside the Terminal window, type this exactly, then press Enter:

cd ~/mu-booking-bot

Nothing visible happens — that's normal. You're now inside the bot folder.

3

Run the bot

Type this and press Enter:

node index.js
4

Watch the Terminal output

Lines will appear showing what the bot is doing. When it's finished you'll see either:

SUCCESS: Booking confirmed — Table 7 | ...
ERROR: ...

You can close Terminal once it prints Browser closed. Done.

📱

If this is the first time running after setup (or after a long break), a Duo Push will appear on your phone. Approve it within 60 seconds — the bot is waiting for you.

Option B — One-click launcher set up once

💡

Create a double-clickable icon for your Dock or Desktop. When you click it, a Terminal window opens and the bot runs with a visible browser. Use this whenever your Mac wasn't plugged in at 10 AM and you need to book manually.

1

Open Automator

Press ⌘ + Space, type Automator, and press Enter.

2

Create a new Application

Click New Document. A window appears asking what type of document to create — click Application, then click Choose.

3

Add a "Run Shell Script" action

You'll see a search box on the left side. Type Run Shell Script and press Enter. When it appears in the list below, double-click it. A text box will appear on the right side of the screen.

4

Paste the command

Click inside the text box on the right, select all the text already there (⌘ + A), and delete it. Then paste this exactly:

osascript -e 'tell application "Terminal" activate do script "export PATH=/opt/homebrew/bin:/usr/local/bin:$PATH; cd ~/mu-booking-bot; node index.js" end tell'
✅

This works on all Macs — Intel and Apple Silicon (M1/M2/M3/M4) — exactly as written. You don't need to change anything.

5

Save it

Press ⌘ + S. Name it Book Table. Save it to your Desktop or Applications folder — wherever feels easiest to find.

6

Add it to your Dock optional

Open Finder, find the Book Table file you just saved, and drag it to your Dock. From now on, one click runs the bot — no Terminal needed.

Checking the logs

To see if it's working

Open Terminal and run:

cat ~/mu-booking-bot/booking.log

Each line is timestamped. Here's what to look for:

[2026-04-17T10:03:22Z] SUCCESS: Booking confirmed for Table 7.
[2026-04-17T10:03:45Z] ERROR: No available slots found for Table 7.
[2026-04-17T10:00:01Z] Skipping — today falls within exception: "Finals Week"

If there was an error, a screenshot named error-[timestamp].png is also saved in the bot folder so you can see what the browser was looking at.

Troubleshooting

Something went wrong? Start here

🔐

"Could not read username from Keychain"

Your credentials weren't saved properly. Re-run bash setup.sh and enter them again.

📱

Duo Push timed out or never arrived

Make sure your phone has signal and Duo notifications are on. Run manually with node index.js --headed so you can watch and approve in time.

🪑

"No available slots found for any table"

Two possibilities:

  • All tables are fully booked — nothing to do, try again tomorrow. The bot already tried Tables 7, 6, 8, 5, 9, and beyond before giving up.
  • It's the quarter registration opening day — the first 4 weeks of tables were just released all at once and need to be booked manually at memorialunion.ucdavis.edu/reservations. Add just that one day as an exception. The bot resumes normally the next morning.
😴

Bot never ran at 10 AM

Two things to check:

  • Wake schedule: Open Terminal and run sudo pmset repeat wake MTWR 09:55:00 to set it, then try again. The Mac must be plugged into power — macOS ignores scheduled wakes on battery, even with the lid open
  • Job registered: Open Terminal and run launchctl list | grep mubooking — if nothing shows, re-run bash setup.sh
🌐

Login is failing / wrong password error

Your UC Davis password may have changed. Re-run bash setup.sh and enter your updated password.

🔄

Duo Push is appearing again after months

The saved session expired (common after summer break). Just approve it once and it saves again — back to skipping after that.

📧

Not getting failure emails

  • Check your spam/junk folder
  • Make sure you used a Gmail App Password, not your regular Gmail password
  • Re-run bash setup.sh to re-enter the email settings
  • Make sure 2-Step Verification is enabled on the Gmail account you're sending from

Keeping the bot working over time

What breaks, why, and how to fix it — no coding experience needed

💡

The bot works by reading the booking website like a person would — clicking buttons and filling in fields. If the website ever changes its layout, the bot might stop finding those buttons. That's the most common reason it breaks. Everything below shows how to fix it.

The most common things that break

🌐

The booking website changes its layout MOST COMMON

The MU occasionally redesigns its website. When this happens, the bot can't find the buttons it's looking for. You'll see an error like "No available slots found" even when you can see slots on the page yourself.

🔑

The UC Davis login page changes

Rare, but occasionally UC Davis updates their login page. The bot might get stuck before even reaching the booking page.

📦

The bot's tools become outdated

Playwright (the tool that controls the browser) and Node.js get updates over time. Once every 6–12 months it's worth refreshing them.

Free tools to have ready

VS

VS Code — free code editor

Download at code.visualstudio.com. This is how you open and edit the bot's files. It highlights code in colour, making it much easier to read.

✦

Claude or ChatGPT — free AI assistant

Available at claude.ai or chatgpt.com. You can paste the error from booking.log and ask it what went wrong and how to fix it. You can also paste sections of index.js and ask it to update them. No coding knowledge needed — just describe what's broken.

Chr

Chrome DevTools — built into Chrome, free

Press ⌘ + Option + I on any webpage in Chrome to open DevTools. You can hover over any element on the page to see its code name — useful for telling an AI what changed.

How to fix it when the website layout changes

The most common fix — takes about 10 minutes with AI help

1

Run the bot manually with the browser visible so you can see exactly where it gets stuck:

node index.js --headed
2

When it gets stuck, open the same page manually in Chrome. Press ⌘ + Option + I to open DevTools. Hover over the button or slot the bot can't find — you'll see its code name highlighted in the panel on the right.

3

Take a screenshot of the DevTools panel showing the element's code. Open claude.ai or chatgpt.com and say something like:

"I have a Playwright bot that books tables on a website. It used to find the table slots with a certain CSS selector but the website changed. Here's what the element looks like now [paste screenshot or HTML]. Here's the relevant section of my index.js [paste the findLatestSlotForTable function]. Can you update the selectors to match the new layout?"
4

The AI will give you updated code. Open index.js in VS Code, find the section it mentioned, and replace it with the new version.

5

Run it again to test:

node index.js --headed

How to update the bot's tools

Do this once every 6–12 months to keep things fresh

Update Playwright and other packages:

cd ~/mu-booking-bot npm update

Update Node.js:

brew upgrade node

After updating Playwright, reinstall its browser:

npx playwright install chromium
💡

If anything breaks after an update, just ask Claude or ChatGPT: "I updated my Playwright bot's dependencies and now it's throwing this error: [paste error]. How do I fix it?"

🛑

Stopping the bot

When you want to turn it off

To permanently disable the daily job, open Terminal and run these two lines:

launchctl unload ~/Library/LaunchAgents/com.mubooking.plist rm ~/Library/LaunchAgents/com.mubooking.plist

To re-enable later, just re-run bash setup.sh.

Passing this to someone new

What the next person needs to do

Share this whole folder with them. They only need to follow these steps:

1

Delete the .chrome-profile/ folder inside the bot folder (this clears the old saved session)

2

Follow all the Setup steps above from Step 1 — bash setup.sh overwrites the old credentials automatically

3

Update exceptions.json with any new quarter dates they want to skip

ℹ️

The blackout dates in exceptions.json carry over automatically — the new person just needs to add any new ones for upcoming quarters.

What's in the folder

In case you're curious

In the download

  • HANDOFF.html This guide — open it in any browser
  • index.js The bot itself — handles login, Duo, booking, and email alerts. Don't edit unless debugging.
  • setup.sh The setup script — run this once to install everything and save your credentials
  • add-exception.js Helper script for adding blackout dates — run with npm run add-exception
  • exceptions.json Blackout dates — the only file you'll edit regularly
  • com.mubooking.plist Tells macOS to run the bot at 10 AM Mon–Thu — managed by setup.sh, don't edit
  • package.json Lists the tools the bot needs — used by setup.sh to install them, don't edit
  • package-lock.json Locks the exact versions of those tools — don't edit

Created by setup.sh

  • node_modules/ The installed tools the bot depends on (Playwright, etc.) — created automatically, don't touch
  • config.json Your email alert settings — written by setup.sh when you enter your email

Created automatically as the bot runs

  • .chrome-profile/ Saved Chrome login session so Duo is only needed once — delete this folder when handing off to someone new
  • booking.log A running log of every booking attempt — one timestamped line per run, success or failure
  • launchd-stdout.log Raw output captured from scheduled runs — useful if something seems wrong and booking.log isn't enough
  • launchd-stderr.log Error output from scheduled runs — usually empty, but check here if the bot isn't running at all
